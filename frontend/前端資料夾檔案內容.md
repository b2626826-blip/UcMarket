# 前端資料夾檔案內容規劃

本文件說明 `frontend/` 正式前端資料夾中，各目錄應該放哪些檔案與內容。  
目前先作為組員分工與後續實作的對照文件，不代表所有檔案都要一次完成。

## 目前定位

`frontend/` 是正式前端實作位置。  
目前根目錄沒有 `公版/` 或 `front/`。若未來重新加入靜態 prototype，只作為畫面參考，不直接當成正式前端搬進來。

前端主要負責：

- 畫面呈現
- 使用者互動
- 表單驗證
- 路由切換
- 登入狀態顯示
- 呼叫後端 REST API

前端不直接連接資料庫，所有資料都要透過後端 API 取得或更新。

## 建議資料夾結構

```text
frontend
├── 排行榜設計語彙.md
├── 前端資料夾檔案內容.md
└── src
    ├── api
    ├── assets
    ├── components
    ├── pages
    │   ├── public
    │   ├── member
    │   └── admin
    ├── router
    ├── store
    └── types
```

## src/pages

`pages` 放「完整頁面」，一個資料夾對應一個主要頁面或路由。  
頁面檔案只負責組合資料、共用元件與頁面流程，不要把 API 細節或大量共用 UI 寫死在頁面裡。

### public

公開頁面，不需要登入即可瀏覽。

| 資料夾 | 對應頁面 | 建議內容 |
|---|---|---|
| `public/home` | 首頁 / 市場列表 | 熱門市場、分類篩選、搜尋、市場卡片列表 |
| `public/market-detail` | 市場詳情 | 市場標題、狀態、Yes/No 價格、交易面板、資料來源、結算規則 |
| `public/login` | 登入 | 帳號密碼表單、登入錯誤提示、導向註冊 |
| `public/register` | 註冊 | 註冊表單、欄位驗證、註冊成功後導向 |

每個公開頁面資料夾之後可放：

```text
index.html 或 Page.jsx
style.css
main.js
README.md
```

若正式採用 React，建議用 `Page.jsx` 或 `index.jsx`。  
若先以純 HTML / CSS / JavaScript 開發，建議用 `index.html`、`style.css`、`main.js`。

### member

會員頁面，需要登入後才能使用。

| 資料夾 | 對應頁面 | 建議內容 |
|---|---|---|
| `member/create-market` | 建立市場 | 市場標題、分類、截止時間、資料來源、結算規則、送出審核 |
| `member/portfolio` | 我的資產總覽 | 餘額、持倉摘要、已實現盈虧、未實現盈虧 |
| `member/wallet` | 錢包頁 | 可用餘額、錢包異動紀錄、交易時間倒序列表 |
| `member/positions` | 持倉頁 | 使用者目前持倉、平均成本、目前盈虧、持倉狀態 |
| `member/trade-history` | 交易紀錄 | 買入 / 賣出紀錄、成交價格、份數、時間 |
| `member/rankings` | 排行榜 | 使用者排行榜、市場排行榜、收益 / 勝率 / 資產排序 |

會員頁面要注意：

- 錢包餘額以後端回傳為準。
- 交易結果與成交價格以後端 API 回傳為準。
- 前端可以顯示試算結果，但不能自行決定最終交易結果。
- 未登入時應導回登入頁或顯示需要登入狀態。

### admin

管理員頁面，需要管理員權限。

| 資料夾 | 對應頁面 | 建議內容 |
|---|---|---|
| `admin/pending-markets` | 待審核市場 | 待審核列表、搜尋、篩選、進入審核詳情 |
| `admin/market-review` | 市場審核 | 市場內容檢查、通過 / 拒絕、審核理由 |
| `admin/market-resolution` | 市場結算 | 選擇結算結果、執行結算、顯示結算狀態 |
| `admin/users` | 使用者管理 | 使用者列表、角色、狀態、停權或恢復 |
| `admin/admin-logs` | 後台操作紀錄 | 管理員操作歷史、時間、動作、目標資料 |

管理員頁面要注意：

- 前端只能控制畫面顯示，真正權限檢查必須由後端執行。
- 審核、結算、停權等操作要有明確確認流程。
- 操作完成後要重新讀取列表，避免畫面狀態與後端不一致。

## src/api

`api` 集中管理後端 API 呼叫。  
頁面不要到處直接寫 `fetch`，避免 API 路徑、錯誤處理與登入 token 管理分散。

建議檔案：

```text
api
├── client.js
├── authApi.js
├── marketApi.js
├── tradeApi.js
├── walletApi.js
├── positionApi.js
├── rankingApi.js
└── adminApi.js
```

各檔案責任：

| 檔案 | 內容 |
|---|---|
| `client.js` | 共用 `fetch` 設定、base URL、JSON 解析、錯誤處理 |
| `authApi.js` | 註冊、登入、登出、查詢目前使用者 |
| `marketApi.js` | 市場列表、市場詳情、建立市場、更新市場 |
| `tradeApi.js` | 交易試算、建立交易 |
| `walletApi.js` | 錢包餘額、錢包異動紀錄 |
| `positionApi.js` | 持倉列表、持倉詳情 |
| `rankingApi.js` | 收益排行、勝率排行、資產排行 |
| `adminApi.js` | 市場審核、結算、使用者管理、操作紀錄 |

## src/router

`router` 管理頁面路由與導頁規則。

建議檔案：

```text
router
├── routes.js
└── guards.js
```

`routes.js` 放路由對應：

| 路由 | 頁面 |
|---|---|
| `/` | `public/home` |
| `/auth/login` | `public/login` |
| `/auth/register` | `public/register` |
| `/markets/:id` | `public/market-detail` |
| `/markets/new` | `member/create-market` |
| `/portfolio` | `member/portfolio` |
| `/wallet` | `member/wallet` |
| `/positions` | `member/positions` |
| `/trades` | `member/trade-history` |
| `/rankings` | `member/rankings` |
| `/admin/markets/pending` | `admin/pending-markets` |
| `/admin/markets/review/:id` | `admin/market-review` |
| `/admin/markets/resolve/:id` | `admin/market-resolution` |
| `/admin/users` | `admin/users` |
| `/admin/logs` | `admin/admin-logs` |

`guards.js` 放導頁判斷：

- 未登入不能進會員頁。
- 非管理員不能進管理員頁。
- 已登入使用者進登入頁時，可導回首頁或會員頁。

## src/store

`store` 管理前端狀態。  
只放跨頁會用到的狀態，不要把單一頁面表單也放進全域狀態。

建議檔案：

```text
store
├── authStore.js
├── walletStore.js
└── uiStore.js
```

| 檔案 | 內容 |
|---|---|
| `authStore.js` | 目前使用者、登入狀態、角色 |
| `walletStore.js` | 錢包摘要、可用餘額、最後更新時間 |
| `uiStore.js` | 全站 loading、錯誤訊息、通知訊息 |

## src/types

`types` 放前後端資料格式說明。  
如果正式前端使用 TypeScript，這裡放 `.ts` type/interface。  
如果先用 JavaScript，這裡可以先用 `.js` 或 Markdown 記錄資料結構。

建議檔案：

```text
types
├── auth.js
├── market.js
├── trade.js
├── wallet.js
├── position.js
├── ranking.js
└── admin.js
```

主要資料型別：

- `User`
- `Market`
- `Trade`
- `Wallet`
- `WalletTransaction`
- `Position`
- `RankingUser`
- `AdminLog`

## src/assets

`assets` 放靜態資源與全站樣式。

建議資料夾：

```text
assets
├── images
├── icons
└── styles
    ├── base.css
    ├── layout.css
    ├── components.css
    └── utilities.css
```

| 檔案 | 內容 |
|---|---|
| `base.css` | reset、字體、全站色彩變數 |
| `layout.css` | 全站版面、container、grid |
| `components.css` | 按鈕、表單、卡片、表格共用樣式 |
| `utilities.css` | 小型工具 class，例如間距、文字顏色、顯示隱藏 |

排行榜頁的視覺細節可參考同層的 `排行榜設計語彙.md`。

## 實作順序建議

1. 先建立 `api/client.js`，統一 API 呼叫方式。
2. 再做 `authApi.js` 與登入 / 註冊頁，讓登入狀態先能運作。
3. 接著做首頁 / 市場列表與市場詳情頁，讓使用者能瀏覽市場。
4. 再做錢包、持倉、交易紀錄與排行榜。
5. 最後做管理員審核、結算、使用者管理與操作紀錄。

## 命名原則

- 資料夾用英文小寫，單字用 `-` 連接。
- 頁面資料夾名稱對應實際路由或功能。
- API 檔案用功能名稱加 `Api`，例如 `marketApi.js`。
- Store 檔案用功能名稱加 `Store`，例如 `authStore.js`。
- 共用元件名稱要看得出用途，例如 `MarketCard`、`TradePanel`。

## 暫時不做的事

目前這份文件只規劃正式前端的檔案內容，不做以下事情：

- 不直接把任何靜態 prototype 搬進 `frontend/`。
- 不新增未確認框架的 package 設定。
- 不先寫大量假功能。
- 不把後端商業邏輯放到前端。
