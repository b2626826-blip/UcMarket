# 排行榜頁設計語彙

> 目前狀態：正式頁面位於 `/rankings`，資料使用 profit、win-rate、assets 三種後端 metric；檔案與品牌已統一為 UcMarket。本文保留黑金視覺語彙，不再代表舊靜態頁的檔名或假資料契約。

本文件保留先前靜態排行榜頁的視覺語言、版面規則與互動邏輯，作為後續細改與組員對照用的設計語彙說明。目前根目錄沒有 `公版/`，正式前端仍以 `frontend/` 為實作位置。

## 頁面定位

排行榜頁是預測市場平台中用來展示使用者表現的資料型頁面。它不是行銷首頁，而是偏向「平台內功能頁」：使用者進來後應該能快速理解誰的收益最高、誰的勝率最好、誰的總資產最高。

頁面重點：

- 展示平台活躍度與競爭感。
- 讓排行榜資料清楚、可比較、可搜尋。
- 保持黑底金色的高級感，但不要犧牲資訊可讀性。
- 作為組員後續實作排行榜功能時的視覺與結構參考。

## 核心風格

目前排行榜頁採用深色金融科技風格：

- 背景以接近純黑的 `#040404` 為主。
- 卡片底色使用 `#090909` 或 `#111`，和背景形成低對比層次。
- 金色 `#d9aa43` 是主要品牌強調色，用於徽章、按鈕、排名、狀態與互動邊框。
- 綠色 `#00d66f` 用於正向收益、勝率、成功或活躍狀態。
- 紅色 `#ff476d` 保留給負向、警示或失敗狀態，目前排行榜頁較少使用。

整體感覺應該是：

- 簡潔
- 冷靜
- 有競技感
- 有資料儀表板感
- 不要太花、不走可愛風、不走一般後台表格風

## 字體與排版

目前共用 CSS 使用 `Inter` 作為主要字體。

文字層級：

- Hero 主標題：超大字重，強調排行榜頁主題。
- 區塊標題：中大型，作為資料區段的入口。
- 卡片數字：大字重，讓關鍵數據一眼可讀。
- 表格文字：以清楚比較為主，避免過度裝飾。
- 次要說明：使用灰色 `#8f8f8f`，降低視覺重量。

注意事項：

- 排行榜頁資料很多，字體應以清楚掃描為優先。
- 英文假名如 `MarketWolf`、`CryptoKing` 可暫留為假資料，但正式版要考慮使用者名稱長度。
- 行距與間距要保持呼吸感，避免資料表太擠。

## 色彩規則

| 用途 | 色彩 | 使用方式 |
|---|---|---|
| 頁面背景 | `#040404` | 全頁主背景 |
| 卡片背景 | `#090909` | 統計卡、前三名卡、表格卡 |
| 次級卡片/輸入 | `#111` | 搜尋框、排名徽章、未選取 tab |
| 主強調色 | `#d9aa43` | 金色按鈕、active tab、排名數字、徽章 |
| 次強調色 | `#b88b2f` | 金色延伸色，必要時用於 hover 或漸層 |
| 正向資料 | `#00d66f` | 收益、活躍狀態、正向百分比 |
| 負向資料 | `#ff476d` | 未來可用於虧損、下降、失敗 |
| 主文字 | `#ffffff` | 標題、主要資料 |
| 次文字 | `#8f8f8f` | 描述、帳號、輔助資訊 |
| 邊框 | `rgba(255,255,255,.06)` | 卡片、分隔線 |

金色只能作為重點，不應整頁大量鋪滿。黑底加金色的比例要維持克制，才會有高級感。

## 頁面結構

目前排行榜頁由以下區塊組成：

1. 全站導覽列
2. 排行榜 Hero
3. 榜單切換 tabs
4. 搜尋列
5. Top 3 使用者卡片
6. 後續排名表格
7. Footer

### 導覽列

導覽列延續先前靜態 prototype 的共用語彙：

- Sticky 固定在上方。
- 黑底透明加 blur。
- 左側品牌識別。
- 中間主功能連結。
- 右側語言、玩法、登入、註冊、選單。

後續品牌統一尚在進行，因此目前文件只記錄語彙，不在此階段強制改名。

### Hero

Hero 目前包含：

- 小徽章：`交易排行榜`
- 主標題：`頂尖預測玩家`
- 描述：說明此頁用來查看收益、勝率與交易活躍度。

設計目的：

- 讓使用者立即知道這是排行榜頁。
- 不要做太大的行銷式 Hero，因為這是功能頁。
- 標題可以大，但下方要很快進入資料。

### 榜單 Tabs

目前 tabs 包含：

- 盈虧榜
- 勝率榜
- 資產榜

設計意圖：

- 使用者可用不同角度看排行榜。
- active 狀態使用金色底、黑色字。
- 未選取狀態使用深灰底、灰色字。
- 三個按鈕使用等寬 grid 排列，避免右側留下不協調的空白。
- 手機版改為單欄排列，讓按鈕可讀且不造成水平捲動。

### Top 3 卡片

前三名以卡片方式獨立展示：

- 第一名置中且稍微放大。
- 第二名與第三名左右排列。
- 第一名使用 crown icon 與金色光暈。
- 每張卡顯示使用者名稱、收益與勝率。

設計目的：

- 強化競技感。
- 讓排行榜不只是表格，也有視覺焦點。
- 第一名應該是整個排行榜區塊中最容易被注意到的人。

注意事項：

- 第一名放大要控制，不可造成版面不穩。
- 手機版應改成單欄排列。
- 未來若使用真實頭像，要保留 fallback icon。

### 後續排名表格

表格欄位目前包含：

- 排名
- 使用者
- 主要市場
- 收益
- 勝率
- 資產

表格用途：

- 提供可掃描、可比較的第 4 名以後資料。
- 搜尋使用者、帳號與市場。
- 配合 tab 切換排序方式。
- 前三名已由上方 Top 3 卡片呈現，因此表格不重複顯示 #1 到 #3。

設計規則：

- 表格包在深色大卡片內。
- 表格上方只保留搜尋入口，不顯示額外標題文字。
- 表頭使用淡金色背景與金色文字。
- row hover 使用非常淡的金色背景。
- 排名數字使用金色。
- 使用者欄位上下排列：名稱在上，帳號在下。
- 狀態使用 pill 標籤。

## 互動語彙

目前 JS 互動包含：

- Tab 切換：以 `profit`、`win-rate`、`assets` 重新取得後端榜單。
- 搜尋：可搜尋使用者名稱、帳號與市場。
- Top 3 連動：切換榜單或搜尋時，前三名卡片會和表格一起更新。
- Hover glow：滑鼠移到卡片或表格時，出現跟隨滑鼠的金色光暈。

互動感覺應該是「精緻但安靜」：

- 不需要大量動畫。
- hover 可以有質感，但不能干擾閱讀。
- 排名資料更新要克制，避免使用者看表格時跳動太劇烈。

## RWD 規則

目前響應式方向：

- 1200px 以下：統計卡改成兩欄，Top 3 改成單欄。
- 768px 以下：Hero 標題縮小，統計卡單欄，表格隱藏部分欄位。

手機版表格目前會隱藏：

- 主要市場
- 勝率
- 資產

保留：

- 排名
- 使用者
- 收益

這個方向合理，因為手機優先看最重要的排名資訊。

## 元件規格

### 卡片

卡片應使用：

- 深色背景
- 低透明白色邊框
- 24px 到 28px 圓角
- hover 上浮
- hover 金色 glow

卡片不應使用：

- 白底
- 大面積純金底
- 過度鮮豔的漸層背景
- 太多陰影造成廉價感

### 按鈕

主要按鈕：

- 金色底
- 黑色字
- 中等圓角或 pill
- hover 可上浮或加 glow

次要按鈕：

- 深灰底
- 灰色字
- hover 轉白

### 搜尋框

搜尋框規則：

- 深灰底
- 無明顯白色外框
- 圓角 14px
- 搜尋按鈕使用金色
- placeholder 使用低對比灰色

### 排名數字

排名數字應：

- 使用 `#4`、`#5` 這種清楚格式，因為前三名已獨立顯示在上方。
- 使用金色與粗體。
- 在 Top 3 可使用 medal/crown 視覺。

## 命名與內容注意事項

舊靜態稿曾出現 Polymarket 品牌、`raking` 拼字與假資料分類；這些不是目前程式碼基準。新增文案或元件時一律沿用 UcMarket 與 `rankings` 命名。

## 後續細改方向

建議下一輪細改可以從以下順序進行：

1. 補上目前登入使用者的個人名次提示。
2. 依真實資料量決定是否增加搜尋與分頁。
3. 只有在後端提供即時事件來源後，才加入收益變動動畫。

## 給組員的實作提醒

這頁不是單純表格，而是「排行榜體驗」：

- 上方統計卡負責建立平台感。
- Top 3 負責建立競技感。
- 表格負責提供完整比較。
- 搜尋與 tabs 負責讓使用者找到自己關心的排名。

實作時請優先保持資訊清楚，再追求裝飾效果。黑底金色是語彙，不是目的；真正目的，是讓使用者快速讀懂誰表現最好。
